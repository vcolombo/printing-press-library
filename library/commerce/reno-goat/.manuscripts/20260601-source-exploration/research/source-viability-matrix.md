# Reno Goat Source Viability Matrix

Date: 2026-06-01
Worktree: printing-press-library PR 855

## Methods Used

- Printing Press `probe-reachability` for page-level transport classification.
- Printing Press `crowd-sniff` for community SDK/code endpoint signals.
- `browser-use` CLI capture for live browser resource evidence where needed.
- Reno Goat `source-probe` for current printed-CLI route status under the plain HTTP transport used by active fanout/model-intel code. Current artifacts capture up to 256KB of response body and flag `body_truncated` when the diagnostic window is capped.
- Direct replay probes only after a replayable endpoint was visible from page/browser evidence.

`curl` alone is not treated as a final method. It is only used here for direct replay and page inspection after the Printing Press reachability/sniff path narrows the candidate.

## Category Targets

Reno Goat is not trying to help a novice source commodity builder supplies such as 2x4s, generic pipe, P-traps, wire nuts, or bulk fasteners. The target is the squishy middle between builder supply and home decor: homeowner-visible selections that commonly need a GC, plumber, electrician, HVAC tech, cabinet installer, or tile/flooring trade to install. Ferguson Home is the model for this sweet spot.

- Appliances: dishwashers, ranges, refrigerators, range hoods, laundry, kitchen packages.
- Plumbing selections: faucets, sinks, vanities, toilets, shower/tub systems, finish valves/trim, bath fixtures.
- Electrical and lighting selections: recessed/downlight fixtures, task/under-cabinet lighting, decorative fixtures, ceiling fans, smart switches/controls, drivers when tied to a fixture system.
- HVAC selections: thermostats, registers/grilles, bath fans, visible HVAC finish components, mini-split/equipment decisions where a homeowner still chooses the product.
- Flooring and surfaces: tile, stone-look/vinyl/laminate flooring, wall tile, floor warming, shower systems, visible surfaces and setting systems when tied to a finish choice.
- Cabinetry and finish hardware: cabinet systems, cabinet/door/bath hardware, rails, trim-adjacent finish pieces, access panels, vents/registers.
- Lower priority / usually out of scope: lumber, framing, commodity pipe/fittings, P-traps, generic fasteners, test tools, raw electrical parts, commodity HVAC service parts, and pro-only wholesale inventory without homeowner-facing product selection.

Current priority source gaps are tracked in `priority-source-gaps.md`. The important open work is appliance/local-showroom breadth (ABW, ADU, AJ Madison, Costco, PC Richard regression/breadth, Appliance Factory regression/breadth, Best Buy regression/breadth, Abt regression/breadth, Homewise Appliance regression/breadth, Spencer's, Grand Appliance, Warners' Stellian) and bath-showroom depth (QualityBath regression/breadth, Vintage Tub regression/breadth, Signature Hardware regression/breadth, HomePerfect, DecorPlanet, Build.com).

## Compound Model Intelligence

The appliance test case showed that broad fan-out search is not enough for high-value installed selections. Reno Goat now has a first-pass `model-intel` workflow that treats installed selections as a compound problem:

- Category discovery: infer installed-selection categories from the query in `--sources auto`, then query replayable source rows such as GE Appliances Searchspring, Bosch category pages, Broan-NuTone ventilation/range-hood model pages, IWAe HVAC cards, 1000Bulbs lighting cards, and other routed active sources; when enabled, readable Brave Search product-result fields can add additional labeled model candidates from retailer/manufacturer pages.
- Routed discovery: with `--sources routed`, reuse Reno Goat's category/room source map so HVAC, electrical, plumbing, flooring, hardware, and appliance sources can participate.
- Model/SKU extraction: normalize model or source SKU candidates from discovered rows and product IDs while filtering generic measurement tokens such as SEER, BTU, lumens, volts, and watts.
- Finish-selection SKU fallback: for categories where the actionable identifier is a retailer product ID or slug rather than an appliance-style model number, preserve source IDs or URL SKU tokens as model-intel selection identifiers.
- Spec enrichment: carry through installation and quick-spec PDF links exposed by source rows, then fetch returned product pages and search-result offer pages to capture linked spec/install/dimmer/warranty PDFs when the retailer exposes them. Known-model enrichment filters sibling-model and promo/legal PDFs.
- Model-page probing: try predictable manufacturer/retailer model pages where appropriate, and report readable, blocked, or missing status instead of silently dropping coverage.
- Search-result fallback: query readable Brave Search result pages and record structured product-result fields as labeled `brave-search:<host>` evidence. Category queries use those fields for additional model/offer candidates; exact-model lookups filter to exact-model title/URL anchors so sibling-model pages do not contaminate pricing. Discovery-only manufacturer rows with no price, such as Broan-NuTone bath fans, are useful only when this exact-model offer pass finds sourceable retailer pricing or when their spec documents are explicitly needed.
- Source-probe triage: run `source-probe --candidate all` to make the WAF/browser gap measurable for appliance and bath showrooms before attempting a new source implementation.
